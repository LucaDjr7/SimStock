# SimStock — Euronext Stock Substitution Engine

> Self-supervised contrastive model that learns stock embeddings on the Euronext universe
> and builds a stock substitution engine — i.e. a ranked list of financially interchangeable
> securities for each ticker.

## Quick Start

### Installation

```bash
cd SimStock/
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Smoke test (sample universe, 1 epoch, CPU-friendly)

```bash
python -m euronext_simstock.main \
  --sample \
  --epochs-per-domain 1 \
  --batch-size 256 \
  --mode cosine \
  --save-name smoke_test
```

### Full run (your own universe)

```bash
python -m euronext_simstock.main \
  --user-csv data/euronext_universe/euronext_universe_for_load_user_universe.csv \
  --epochs-per-domain 5 \
  --batch-size 512 \
  --mode dtw \
  --dtw-reduction pca \
  --dtw-sim-method rank \
  --target-avg-substitutes 10 \
  --save-name euronext_simstock_v1
```

The universe CSV must have at minimum a `ticker` column with Yahoo Finance-style tickers
(`AIR.PA`, `ASML.AS`, `ENI.MI`, …). Supported Euronext suffixes: `.PA`, `.AS`, `.BR`,
`.LS`, `.IR`, `.OL`, `.MI`.

### Resuming after interruption

Checkpoints are saved after every epoch to:

```
artifacts/models/checkpoints/<save_name>/<save_name>_last.pt
```

Resume with:

```bash
python -m euronext_simstock.main \
  --user-csv data/euronext_universe/euronext_universe_for_load_user_universe.csv \
  --resume-checkpoint artifacts/models/checkpoints/euronext_simstock_v1/euronext_simstock_v1_last.pt \
  --save-name euronext_simstock_v1
```

For long runs on Linux, use `tmux`:

```bash
tmux new -s simstock
python -m euronext_simstock.main --user-csv data/euronext_universe/euronext_universe_for_load_user_universe.csv --mode dtw
# detach: Ctrl-b then d  |  reattach: tmux attach -t simstock
```

## Architecture

### Step 1 — Universe & OHLCV data

**Files:** `euronext_simstock/data/universe.py`, `euronext_simstock/data/loader.py`

- Input: user CSV (minimum column: `ticker`) or built-in sample (~70 tickers).
- Download: chunks of 50 tickers, exponential retry (3 attempts), Parquet cache indexed by
  `hash(tickers)+dates` to avoid cross-universe collisions, partial cache for network
  interruptions.
- Sector metadata from `yfinance.Ticker.info`, stored incrementally.

### Step 2 — Feature Engineering & Panelisation

**Files:** `euronext_simstock/data/preprocessing.py`, `euronext_simstock/config.py`

Default mode (`feature_mode="paper"`) — two-pass computation:

1. **5 daily z-features:**
   ```
   zOpen   = Open_t  / Close_t  − 1
   zHigh   = High_t  / Close_t  − 1
   zLow    = Low_t   / Close_t  − 1
   zClose  = Close_t / Close_{t-1} − 1
   zVolume = Vol_t   / Vol_{t-1}   − 1
   ```

2. **Temporal variants:** trailing moving average over `[5, 10, 15, 20, 25]` days for each
   z-feature → **25 features total** (`data_size = 5 × 5`).

Tickers with fewer than 200 days of history are dropped.

**Temporal domains:** the training panel is split into domains `D_1 … D_T` of 126 trading
days (~6 months):

```python
TimeWindow:
  train_start     = "2019-01-01"  →  train_end     = "2023-12-31"  # training
  reference_start = "2024-01-01"  →  reference_end = "2024-12-31"  # embedding extraction
  test_start      = "2025-01-01"  →  test_end      = "2025-12-31"  # out-of-sample
```

### Step 3 — SimStock Model

**Files:** `euronext_simstock/models/simstock.py`, `euronext_simstock/models/layers.py`

Four blocks:

#### a) DRAIN-LSTM — dynamic parameter generator
Gaussian noise `z ∈ ℝ^{25}` initialises an LSTM (`hidden_dim = latent_dim = 25`). At each
temporal domain the LSTM produces vector `E` encoding the **attention weights** for that domain:
```
z  →  LSTM(latent=25)  →  lin_down(25 → 128 → e_dim)  →  E
E  →  lin_up(e_dim → 128 → 25)  →  next LSTM input
```
`e_dim = (n_tokens × 3 + TOKEN_DIM) × TOKEN_DIM + TOKEN_DIM` ≈ 198 000 scalars.

#### b) NumericalEmbedder — tokeniser
Each scalar feature `x_j` is projected to a 256-dim token:
```
token_j = x_j * W_j + b_j   # W_j, b_j ∈ ℝ^{256}, learned
```
A learned `[CLS]` token is prepended → sequence of 26 tokens × 256 dims.

#### c) DRAIN Attention (parameterised by E)
`E` is reshaped into `Q, K, V, W_out` matrices for multi-head attention (4 heads, 64 dims
each). Attention weights **change at every temporal domain**. Output: `[CLS]` token after
LayerNorm + residual connection.

#### d) Sector Embedding
```python
sector_vec = sector_projection(Sector_embedding(sector_id))   # ℝ^{256} → ℝ^{25}
X = X + sector_vec   # added before tokenisation
```

**Key parameters:**
```python
ModelConfig:
  data_size       = 25      # = n_features
  TOKEN_DIM       = 256
  N_HEADS         = 4
  hidden_dim      = 128
  sector_emb      = 256
  triplet_margin  = 0.4
  uniformity_weight = 0.01
```

### Step 4 — Training

**File:** `euronext_simstock/training/train.py`

Sequential domain-by-domain training. The state `(E, hidden)` from domain `D_k` is passed as
initialisation to `D_{k+1}`, encoding temporal continuity in the attention parameters.

**Loss:**
```
L = triplet_loss + 0.01 × uniformity_loss

triplet_loss = mean( softplus( d_pos − d_neg + margin ) )
d_pos = || normalize(anchor) − normalize(positive) ||₂
d_neg = || normalize(anchor) − normalize(negative) ||₂
```

**Augmentations:**
- **Positive** (gentle perturbation): `λ·X + (1−λ)·X[perm]` with `λ = 0.7`
- **Hard negative** (`negative_mode="in_batch_hard"`): nearest neighbour of the anchor among
  normalised batch embeddings
- **Dimension negative** (`negative_mode="dimension"`): strong corruption `(1−λ)·X + λ·X[perm]`

**Uniformity loss** (prevents embedding collapse):
```python
log( mean( exp(−2 · ||z_i − z_j||²) ) )   # over all off-diagonal pairs
```

**Training parameters:**
```python
TrainingConfig:
  epochs_per_domain = 5
  batch_size        = 512
  learning_rate     = 1e-3
  grad_clip         = 1.0
  seed              = 42
```

### Step 5 — Embedding Extraction & Similarity

**Files:** `euronext_simstock/training/train.py`, `euronext_simstock/similarity/`

The trained model (with `E_final, hidden_final`) runs in `eval()` mode over the reference
period (2024), producing daily per-ticker embeddings (256 dims).

**Four similarity modes:**

| Mode | Description |
|---|---|
| `dtw` | PCA 256→1D compression, then parallel `fastdtw` on all pairs, rank-based similarity |
| `cosine` | Cosine on mean embedding per ticker, rescaled to `[0, 1]` |
| `l2rank` | L2 distance on mean embedding, converted to global rank |
| `snapshot` | Cosine on the last available daily embedding |

**Threshold calibration:**
```python
threshold = quantile(sim_vals, 1 − target_avg_substitutes / (n_tickers − 1))
```
Default target: 10 substitutes per ticker on average.

## Outputs & Deployment

Artifacts are written to `artifacts/embeddings/<save_name>_*`:

| File | Content |
|---|---|
| `_daily.parquet` | Per-ticker daily embeddings (ticker, date, e_0…e_255) |
| `_dtw_simmat.parquet` | N×N similarity matrix (if `--mode dtw`) |
| `_engine.npz` | Serialised `SubstitutionEngine` for deployment |
| `_substitution_table.parquet` | Wide format (1 row per ticker, comma-separated substitutes) |
| `_substitution_long.parquet` | Long format (1 row per pair, SQL-friendly) |
| `_similarity_diagnostics.parquet` | Threshold impact table for calibration |
| `_run_meta.json` | Hyperparameter traceability |

**SubstitutionEngine API:**
```python
from euronext_simstock.similarity.substitution import SubstitutionEngine

engine = SubstitutionEngine.load("artifacts/embeddings/euronext_simstock_v1_engine.npz")

engine.all_substitutes("AIR.PA", threshold=0.85)
# → [("SAF.PA", 0.92), ("DG.PA", 0.88), ...]

engine.top_k_substitutes("AIR.PA", k=5)
engine.are_substitutes("AIR.PA", "SAF.PA")
engine.substitution_groups()            # connected components (union-find)
engine.mutual_substitution_cliques()    # maximal cliques (networkx)
engine.compare_trades(portfolio_x, portfolio_y)   # greedy portfolio matching
```

**Training metrics to monitor:**

| Metric | Expected behaviour |
|---|---|
| `triplet_loss` | Decreasing across domains |
| `d_pos` | Low (positives close to anchor) |
| `d_neg` | High (`d_neg > d_pos + margin`) |
| `uniformity_loss` | Negative and stable (no collapse) |

## Dependencies

| Library | Role |
|---|---|
| `torch ≥ 2.5` | Neural network, GPU/CPU training |
| `einops` | Tensor manipulation in attention blocks |
